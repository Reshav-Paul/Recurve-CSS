Default styles for HTML elements

The default styling of html elements has been given an upgrade so that they look much more modern without the need for writing extra css unless the customization is required.
Other than the appearance overhaul, there are a few other style changes like line-height and font size for headings, etc.

Grid System

Containers

Containers are the building blocks of the grid layout. They contain the rows that construct the grid.
They also contain your content so that you can apply the required margin, padding and alignment.

  • The class .container is responsive across all viewports and sets a max-width at each breakpoint
  • The class .container-fluid is always 100% wide with respect to its parent
  • container-[breakpoint] are 100% wide upto the specified breakpoint
<div class = "container-xl"> 100% wide until extra large breakpoint </div>
<div class = "container-lg"> 100% wide until large breakpoint </div>
<div class = "container-md"> 100% wide until medium breakpoint </div>
<div class = "container-sm"> 100% wide until small breakpoint </div>

Rows and Columns

Responsive colums

Classes .col--[breakpoint]-[size] can be used to change the column width at each breakpoint.
Since recurve has a desktop-first design approach, the default .col-[responsive]-[size] classes specify the size of the column for the specified breakpoint and downwards
Responsive columns can be mixed with .col columns and auto-width columns to achieve the required design.

Responsive Auto width Columns

.col-auto and col-[breakpoint]-auto classes can be used to create auto-width columns which have width: auto
.col-[breakpoint]-auto makes the width auto for screens of that size and smaller, while .col-auto is the default and applies for all screens.

No Gutters

The gutters between columns can be removed with .no-gutter. This removes the horizontal negative margins from .row and the horizontal padding from all its immediate children columns.

Ordering items

The columns in a row can be ordered with .order-[number] or with the responsive variants .order-[breakpoint]-[number] which order the elements at the specified breakpoints and downwards.
